# Synapse — 对外 Demo / 比赛展示：完整功能范围（本地数据）

**原则**：评委与观众看到的是「闭环、可点、可讲清故事」；数据以 **localStorage + 少量服务端 JSON（如 `data/users.json`）** 为主，**不接数据库**。

---

## 一、当前已具备（可写进路演脚本）

| 能力 | 说明 |
|------|------|
| 角色分流 | 商务 / 优化师 / 管理员，登录后首页与权限不同 |
| 需求全流程 | 新建三步（原话 → AI 解析 → 确认）、DRAFT 预审、AI 评估、对话流式 + 取消 + 带对话重评、提交优化师 |
| 参考案例建需求 | 经验库案例详情 →「参考此案例创建需求」→ 预填客户原话（`?refCase=`） |
| 经验库 | 筛选、搜索、案例详情、**分享新案例**（写入 localStorage） |
| 项目看板 | 聊天、任务、投放摘要（模拟数据）、图表、**商务充值** |
| 首页 | 财务汇总、项目消耗/ROI 排名（基于本地项目字段） |
| AI | M1 解析 / M2 评估 / 对话 SSE；DeepSeek 优先、否则 Gemini |

---

## 二、建议补齐的「完整 Demo」项（仍全部本地）

以下不依赖数据库，但能明显提升「完整度」与评委观感。

### 1. 叙事与引导（高优先级）— **已实现**

- **Demo 模式一键填充**：登录页底部「**重置为演示数据**」→ 调用 `seedDemoDataToLocalStorage()`（`lib/demo-seed.ts`），写入 Mock 全套数据。
- **路演时间轴**：首页顶部可折叠 **「比赛 / 对外 Demo · 3 分钟演示路线」**（`components/dashboard/DemoTourBanner.tsx`）。
- 首次进入空状态插画：仍为可选增强。

### 2. 通知与「像真系统」的细节 — **已实现**

- **`pushLocalNotification` + `synapse-notifications-updated` 事件**：`addNotification` / 已读后广播，**TopBar 铃铛未读角标**实时更新。
- 触发点：新建需求草稿、异步 AI 评估完成、提交优化师、接单/拒绝、追问、项目充值、经验库分享案例。

### 3. 经验库与需求的「闭环话术」— **部分实现**

- 需求详情 **「相似经验案例」**（规则匹配行业/地区/媒体，`lib/similar-knowledge.ts`），点击打开 `CaseDetail`。
- 案例卡片短文案：可选增强。

### 4. 导出与展示（比赛加分）— **部分实现**

- 需求详情 **「复制摘要」**：`buildRequirementMarkdown`（`lib/requirement-markdown.ts`）→ 剪贴板。
- 项目「生成报告」：原有能力保留，未改。

### 5. 稳定性与演示保险 — **已实现（可选开关）**

- 服务端 **`DEMO_SAFE_MODE=1`**（见 `.env.example`）：`/api/ai/parse` 与 `/api/ai/evaluate` 在模型抛错时返回 `demo-ai-fallback` 本地结果。
- 环境检测页：仍为可选。

### 6. 视觉与移动端

- **关键路径在 375px 走一遍**：仍为建议手工回归。
- **加载态**：未统一，可后续优化。

---

## 三、刻意不做（本轮范围外）

- 真实数据库、多租户服务端持久化（除注册用的 `data/users.json` 可保留）。
- 真实广告平台 API、真实财务对账。
- 生产级密码哈希与复杂权限模型（Demo 可继续「邮箱匹配 + 任意密码」并**在登录页注明演示用途**）。

---

## 四、路演时可讲的「完整故事」模板（1 句话对应一块屏）

1. **痛点**：商务与优化师信息反复、术语鸿沟、经验不沉淀。  
2. **Synapse**：同一套界面，AI 把原话变结构化需求 + 可解释评估。  
3. **经验库**：沉淀案例，**参考案例一键开需求**；**人人可分享新案例**。  
4. **协同**：需求状态机 + 项目看板 + 模拟投放摘要 + **充值记录**。  
5. **落地**：当前 Demo 全本地，下一步接库与真实投放 API。
